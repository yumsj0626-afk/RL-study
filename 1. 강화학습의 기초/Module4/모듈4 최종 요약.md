## 최종 요약: Policies, Value Functions & Bellman Equations

---

### 정책 (Policy)

- **결정론적 정책**: 각 상태에서 하나의 행동을 확정적으로 선택한다

$$\pi(s) = a$$

- **확률적 정책**: 각 상태에서 행동에 대한 확률 분포를 따른다

$$\pi(a|s) = P(A=a \mid S=s)$$

- 정책은 반드시 **현재 상태에만** 의존한다. 시간이나 이전 상태에 의존할 수 없다.
- 이것은 에이전트의 한계가 아니라 **상태(State)에 대한 요구사항**이다.

---

### 가치 함수 (Value Functions)

가치 함수는 무한한 미래의 가능성을 **하나의 숫자로 압축**한다.

| | 정의 | 수식 |
| :--- | :--- | :--- |
| 상태 가치 함수 | 상태 $s$에서 $\pi$를 따를 때 기대 누적 보상 | $V_\pi(s) = \mathbb{E}_\pi[G_t \mid S_t=s]$ |
| 행동 가치 함수 | 상태 $s$에서 행동 $a$ 선택 후 $\pi$를 따를 때 기대 누적 보상 | $Q_\pi(s,a) = \mathbb{E}_\pi[G_t \mid S_t=s, A_t=a]$ |

---

### 벨만 방정식 (Bellman Equations)

현재 가치를 미래 가치로 재귀적으로 표현한다.

$$V_\pi(s) = \sum_a \pi(a|s) \sum_{s',r} p(s',r \mid s,a)\left[r + \gamma V_\pi(s')\right]$$

$$Q_\pi(s,a) = \sum_{s',r} p(s',r \mid s,a)\left[r + \gamma \sum_{a'} \pi(a'|s') Q_\pi(s',a')\right]$$

- 연립방정식으로 변환해 직접 풀 수 있다.
- 단, 상태 수가 많으면 현실적으로 불가능하다.

---

### 최적 정책과 최적 가치 함수

- **최적 정책 $\pi^*$**: 모든 상태에서 가장 높은 가치를 달성하는 정책이다. 항상 최소 하나는 존재한다.
- **최적 가치 함수**: 모든 최적 정책이 동일한 $V^*$와 $Q^*$를 공유한다.

$$V^*(s) = \max_\pi V_\pi(s)$$

**벨만 최적 방정식**: 정책 참조 없이 $\max_a$로 대체한다.

$$V^*(s) = \max_a \sum_{s',r} p(s',r \mid s,a)\left[r + \gamma V^*(s')\right]$$

$$Q^*(s,a) = \sum_{s',r} p(s',r \mid s,a)\left[r + \gamma \max_{a'} Q^*(s',a')\right]$$

- $\max$ 연산 때문에 **비선형**이다. 선형대수로 직접 풀 수 없다.

---

### 최적 가치 함수 → 최적 정책 도출

| 보유한 것 | 방법 | 난이도 |
| :--- | :--- | :--- |
| $V^*$ | 각 상태에서 1 스텝 룩어헤드 후 $\arg\max_a$ | 환경 동역학 $p$ 필요 |
| $Q^*$ | 바로 $\arg\max_a Q^*(s,a)$ 선택 | 룩어헤드 불필요, 더 간단 |

---

### 전체 흐름 한눈에 보기
```
정책(π) 정의
    ↓
가치 함수 계산 (V_π, Q_π)
    ↓
벨만 방정식으로 구조화
    ↓
최적 가치 함수 (V*, Q*) 탐색
    ↓
최적 정책 (π*) 도출
```

다음 주에는 벨만 방정식을 기반으로 **최적 정책을 실제로 계산하는 알고리즘**을 다룬다.
