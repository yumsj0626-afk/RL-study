## 1-1 Specifying Policies (정책의 정의)

### 핵심 개념 요약

| 원문 | 번역 | 포인트 |
| :--- | :--- | :--- |
| A policy maps each state to a probability distribution over actions | 정책은 각 상태를 행동의 확률 분포에 매핑한다 | 정책 = 상태 → 행동 확률 분포 |
| Deterministic policy: Pi(S) → single action | 결정론적 정책: 각 상태에서 하나의 행동만 선택한다 | 확률 1로 하나의 행동 선택 |
| Stochastic policy: Pi(a\|s) → probability distribution | 확률적 정책: 여러 행동이 0이 아닌 확률로 선택될 수 있다 | 탐색(Exploration)에 유용 |
| Policy must depend only on the current state | 정책은 반드시 현재 상태에만 의존해야 한다 | 시간, 이전 상태, 이전 행동 등은 사용 불가 |

---

### 결정론적 정책 (Deterministic Policy)

가장 단순한 형태의 정책이다.

각 상태 $s$에 대해 **하나의 행동**을 정확히 지정한다.

$$\pi(s) = a$$

- 표기법: $\pi(s)$ → 상태 $s$에서 정책 $\pi$가 선택하는 행동
- 예시: 그리드 맵에서 집을 향해 이동하는 화살표처럼, 각 위치(상태)마다 방향(행동)이 하나로 고정되어 있다
- 여러 상태에서 같은 행동을 선택할 수 있다
- 어떤 행동은 아무 상태에서도 선택되지 않을 수 있다

---

### 확률적 정책 (Stochastic Policy)

각 상태에서 행동을 **확률적으로** 선택하는 정책이다.

$$\pi(a|s) = P(\text{action} = a \mid \text{state} = s)$$

유효한 정책이 되려면 두 가지 조건을 만족해야 한다.

- **조건 1**: 모든 행동의 확률 합 = 1

$$\sum_{a} \pi(a|s) = 1, \quad \forall s$$

- **조건 2**: 모든 행동의 확률은 0 이상

$$\pi(a|s) \geq 0, \quad \forall a, s$$

> 예시: 그리드 맵 하단 행에서 "위" 또는 "오른쪽"을 각각 50% 확률로 선택하는 정책이다.
> 이 경우에도 집까지 도달하는 스텝 수는 결정론적 정책과 동일하다.

---

### 확률적 정책이 왜 필요한가?

이전에 배운 $\epsilon$-greedy 정책처럼, 확률적 정책은 **탐색(Exploration)** 에 유용하다.

MDP에서도 동일한 **탐색-활용 트레이드오프(Exploration-Exploitation Trade-off)** 가 존재한다.

---

### 정책의 핵심 제약 조건

정책은 반드시 **현재 상태(Current State)에만** 의존해야 한다.

다음은 허용되지 않는다.

- 시간(Time)에 의존하는 정책
- 이전 상태(Previous States)에 의존하는 정책
- 이전 행동(Previous Actions)에 의존하는 정책

> 예시: "지난번과 반대 방향으로 이동한다 (좌↔우 교대)" 는 **유효하지 않은 정책**이다.
> 이전 행동에 조건이 걸려 있기 때문이다.

이것은 에이전트의 한계가 아니라 **상태(State)에 대한 요구사항**으로 이해해야 한다.

만약 이전 행동이 의사결정에 필요하다면, **그 정보를 상태에 포함**시키면 된다.

MDP에서 상태는 의사결정에 필요한 **모든 정보를 포함**한다고 가정한다.

---

### 핵심 정리

- 정책은 상태를 행동의 확률 분포에 매핑하는 함수이다
- 결정론적 정책은 각 상태에서 하나의 행동을 선택하고, 확률적 정책은 여러 행동을 확률적으로 선택한다
- 정책은 **현재 상태에만** 의존해야 하며, 시간이나 이전 상태/행동에는 의존할 수 없다

## 1-2 Value Functions (가치 함수)

### 핵심 개념 요약

| 원문 | 번역 | 포인트 |
| :--- | :--- | :--- |
| State value function: expected return from a given state | 상태 가치 함수: 특정 상태에서 시작했을 때 기대할 수 있는 누적 보상 | $V_\pi(s)$ |
| Action value function: expected return after selecting action A, then following Pi | 행동 가치 함수: 행동 A를 선택한 후 정책 Pi를 따를 때의 기대 누적 보상 | $Q_\pi(s, a)$ |
| Value functions are defined with respect to a policy | 가치 함수는 반드시 특정 정책을 기준으로 정의된다 | 정책이 바뀌면 가치 함수도 바뀐다 |
| Value functions summarize all possible futures by averaging over returns | 가치 함수는 모든 가능한 미래를 평균내어 하나의 숫자로 요약한다 | 즉시 보상 대신 장기적 품질을 평가 |

---

### 왜 가치 함수가 필요한가?

강화학습의 목표는 **장기적으로 가장 많은 보상**을 얻는 정책을 학습하는 것이다.

그런데 실제 Return은 두 가지 문제가 있다.

- **문제 1**: Return은 에피소드가 끝나야만 알 수 있다 (즉시 사용 불가)
- **문제 2**: 정책과 환경의 확률성(Stochasticity) 때문에 Return이 매번 달라진다

가치 함수는 이 두 문제를 해결한다.

> 현재 상황의 품질을 **즉시** 평가할 수 있도록, 모든 가능한 미래를 평균내어 하나의 숫자로 요약한다.

---

### 상태 가치 함수 (State Value Function)

$$V_\pi(s) = \mathbb{E}_\pi \left[ G_t \mid S_t = s \right]$$

- 상태 $s$에서 시작해서 정책 $\pi$를 따를 때 기대할 수 있는 누적 보상이다
- 아래 첨자 $\pi$는 이 가치 함수가 정책 $\pi$에 종속됨을 나타낸다
- 정책이 달라지면 같은 상태라도 가치가 달라진다

---

### 행동 가치 함수 (Action Value Function)

$$Q_\pi(s, a) = \mathbb{E}_\pi \left[ G_t \mid S_t = s, A_t = a \right]$$

- 상태 $s$에서 행동 $a$를 먼저 선택하고, 이후 정책 $\pi$를 따를 때의 기대 누적 보상이다
- 각 행동의 품질을 비교할 수 있어, 어떤 행동이 더 좋은지 직접 평가 가능하다

---

### 체스 예시로 이해하기

체스는 대표적인 **에피소드형 MDP**이다.

| 요소 | 내용 |
| :--- | :--- |
| State | 체스판 위 모든 말의 위치 |
| Action | 가능한 모든 합법적 이동 |
| Reward | 승리 시 $+1$, 그 외 모든 스텝 $0$ |
| Termination | 승/패/무 중 하나로 게임 종료 |

보상만으로는 게임 중 에이전트가 얼마나 잘 하고 있는지 알 수 없다.

게임이 끝날 때까지 기다려야 $+1$을 받는지 알 수 있기 때문이다.

반면 **상태 가치 함수**는 훨씬 더 많은 정보를 제공한다.

> 유일한 비영 보상이 승리 시 $+1$이므로, $V_\pi(s)$는 현재 상태에서 정책 $\pi$를 따를 때 **이길 확률**과 동일하다.

상대방의 이동도 상태 전이의 일부이다.

상대가 말을 움직여 새 상태 $s'$이 되면, $V_\pi(s') < V_\pi(s)$ 일 수 있다.

즉, 현재 상태보다 이길 확률이 낮아진 것이다.

---

### 그리드 월드 예시로 이해하기

간단한 **Continuing MDP**로 가치 함수의 직관을 쌓아보자.

| 요소 | 내용 |
| :--- | :--- |
| State | 그리드 위 각 위치 |
| Action | 상/하/좌/우 이동 |
| Reward (일반) | 대부분의 이동: $0$ |
| Reward (벽 충돌) | $-1$ |
| Reward (상태 A) | 모든 행동: $+10$, 이후 A'으로 이동 |
| Reward (상태 B) | 모든 행동: $+5$, 이후 B'으로 이동 |
| 정책 | 균등 랜덤 정책 (Uniform Random Policy) |
| $\gamma$ | $0.9$ |

**주요 관찰 포인트**

- 그리드 하단 모서리 근처는 **음수 가치**를 가진다
  → 랜덤 정책은 A, B에 도달하기 전에 벽에 자주 충돌하기 때문이다

- 상태 A의 가치는 $10$보다 낮다
  → A에서 모든 행동이 $+10$을 주지만, A'은 하단 벽 근처라 충돌 확률이 높기 때문이다

- 상태 B의 가치는 $5$보다 약간 높다
  → B'은 그리드 중앙이라 벽 충돌 가능성이 낮고, 고가치 상태 A, B와 가깝기 때문이다

---

### 핵심 정리

- **상태 가치 함수 $V_\pi(s)$**: 상태 $s$에서 정책 $\pi$를 따를 때 기대되는 누적 보상이다
- **행동 가치 함수 $Q_\pi(s, a)$**: 상태 $s$에서 행동 $a$를 선택한 후 정책 $\pi$를 따를 때 기대되는 누적 보상이다
- 두 함수 모두 **특정 정책에 종속**되며, 정책이 달라지면 가치도 달라진다
- 가치 함수는 장기적 결과를 즉시 평가할 수 있게 해주어, 정책의 품질을 비교하는 데 핵심적인 역할을 한다

## 2-1 Bellman Equation (벨만 방정식)

### 왜 필요한가?

자전거 타다 돌에 걸려 균형을 잃었을 때 — 다치지 않았어도 그 상태가 나쁘다는 걸 안다.

강화학습도 마찬가지다. 미래 보상을 끝까지 다 보지 않아도, **현재 상태의 가치를 미래 상태의 가치로 표현**할 수 있다.

---

### 상태 가치 함수의 벨만 방정식

Return의 재귀적 표현 $G_t = R_{t+1} + \gamma G_{t+1}$ 을 가치 함수 정의에 대입하면 아래가 나온다.

$$V_\pi(s) = \sum_a \pi(a|s) \sum_{s', r} p(s', r \mid s, a) \left[ r + \gamma V_\pi(s') \right]$$

> 현재 상태의 가치 = 즉각 보상 + 할인된 다음 상태 가치의 가중 평균

정책 $\pi$와 환경 $p$ 모두 시간에 의존하지 않기 때문에, $V_\pi(s')$로 재귀 대체가 가능하다.

---

### 행동 가치 함수의 벨만 방정식

행동이 이미 고정되어 있으므로, 정책으로 행동을 고르는 단계 없이 바로 환경 $p$를 적용한다.

$$Q_\pi(s, a) = \sum_{s', r} p(s', r \mid s, a) \left[ r + \gamma \sum_{a'} \pi(a'|s') Q_\pi(s', a') \right]$$

$V_\pi$와 달리, 다음 상태 $s'$에서 다시 행동 $a'$을 선택하는 과정까지 포함해야 재귀가 완성된다.

---

### 핵심 정리

- 벨만 방정식 = **현재 가치를 미래 가치로 재귀 표현**한 것
- 가치 함수 덕분에 무한한 미래를 하나의 숫자로 압축할 수 있다
- 이 구조가 나중에 가치 함수를 효율적으로 추정하는 알고리즘의 기반이 된다
---
## 2-2 Why Bellman Equations? (벨만 방정식을 왜 쓰는가?)

### 핵심 아이디어

벨만 방정식 없이는 가치 함수를 구하려면 **무한히 많은 미래 경우의 수**를 다 따져야 한다.

벨만 방정식은 이 무한한 합을 **연립 선형 방정식 문제로 축소**시켜준다.

---

### 그리드 예시로 이해하기

4개의 상태 A, B, C, D로 구성된 그리드이다.

| 요소 | 내용 |
| :--- | :--- |
| Action | 상/하/좌/우 (벽에 막히면 제자리) |
| Reward | 상태 B에 도달하면 $+5$, 나머지 $0$ |
| 정책 | 균등 랜덤 (각 방향 25%) |
| $\gamma$ | $0.7$ |

상태 A에 벨만 방정식을 적용하면 아래처럼 쓸 수 있다.

$$V_\pi(A) = \frac{1}{4}[5 + \gamma V_\pi(B)] + \frac{1}{4}[0 + \gamma V_\pi(C)] + \frac{1}{2}[0 + \gamma V_\pi(A)]$$

- 오른쪽 → B 도착, 보상 $+5$ (확률 1/4)
- 아래 → C 도착, 보상 $0$ (확률 1/4)
- 위 or 왼쪽 → A 제자리, 보상 $0$ (확률 1/2로 합산)

나머지 B, C, D도 같은 방식으로 쓰면 **4개 변수, 4개 방정식**의 연립방정식이 완성된다.

이걸 풀면 각 상태의 가치를 구할 수 있다.

---

### 한계도 있다

이 방법은 **상태 수가 적을 때**만 현실적이다.

체스의 경우 가능한 상태가 약 $10^{45}$개이다. 연립방정식 자체를 세우는 것도 불가능하다.

→ 이후 강의에서 대규모 문제에 적용 가능한 벨만 방정식 기반 알고리즘을 다룬다.

---

### 핵심 정리

- 벨만 방정식 = **무한한 미래 합 → 연립 선형 방정식**으로 변환
- 작은 MDP에서는 직접 풀어서 가치 함수를 구할 수 있다
- 대규모 문제에서는 이를 기반으로 한 근사 알고리즘이 필요하다

## 3-1 Optimal Policies (최적 정책)

### 정책 비교: 언제 더 좋다고 할 수 있나?

정책 $\pi_1$이 $\pi_2$보다 좋다 = **모든 상태에서** $\pi_1$의 가치가 $\pi_2$보다 크거나 같다.

$$\pi_1 \geq \pi_2 \iff V_{\pi_1}(s) \geq V_{\pi_2}(s), \quad \forall s$$

어떤 상태에서는 $\pi_1$이, 다른 상태에서는 $\pi_2$가 더 높다면 → 우열을 가릴 수 없다.

---

### 최적 정책 $\pi^*$

> 모든 상태에서 다른 어떤 정책보다도 가치가 크거나 같은 정책이다.

- 항상 최소 하나는 존재한다
- 여러 개일 수도 있다
- $\pi^*$로 표기한다

**왜 항상 존재하는가?**

$\pi_1$이 잘하는 상태 + $\pi_2$가 잘하는 상태를 합쳐서,
각 상태에서 더 높은 쪽을 선택하는 $\pi_3$를 만들면 → $\pi_3 \geq \pi_1$, $\pi_3 \geq \pi_2$ 가 항상 성립한다.

즉, **한 상태에서 잘하기 위해 다른 상태를 희생할 필요가 없다.**

---

### 예시: Two-Choice MDP

상태 X에서만 선택이 있다.

| 행동 | 결과 |
| :--- | :--- |
| $A_1$ | 즉시 보상 $+1$ → 상태 Y → 다시 X |
| $A_2$ | 한 스텝 후 보상 $+2$ → 상태 Z → 다시 X |

**$\gamma = 0$ 일 때**

미래 보상은 무시한다. $A_1$의 즉시 보상 $+1 > A_2$의 즉시 보상 $0$ → $\pi_1$ 최적

**$\gamma = 0.9$ 일 때**

$$V_{\pi_1}(X) = \frac{1}{1-\gamma^2} \approx 5.3$$

$$V_{\pi_2}(X) = \frac{2\gamma}{1-\gamma^2} \approx 9.5$$

$9.5 > 5.3$ 이므로 → $\pi_2$ 최적

> 할인율이 높을수록 미래 보상이 중요해지므로, 최적 정책이 바뀔 수 있다.

---

### 브루트 포스로 최적 정책을 찾으면?

결정론적 정책의 수 = $|A|^{|S|}$ (행동 수의 상태 수 제곱)

상태/행동 수가 조금만 커져도 탐색이 불가능해진다.

→ 다음 강의에서 **벨만 최적 방정식**을 통한 효율적인 방법을 다룬다.

---

### 핵심 정리

- 최적 정책 = **모든 상태에서** 가장 높은 가치를 가지는 정책이다
- 항상 최소 하나는 존재하며, 여러 개일 수 있다
- 브루트 포스 탐색은 현실적으로 불가능 → 벨만 최적 방정식이 필요하다

---
## 3-2 Optimal Value Functions (최적 가치 함수)

### 최적 가치 함수란?

최적 정책 $\pi^*$의 가치 함수는 **모든 상태에서 가능한 가장 높은 가치**를 가진다.

$$V^*(s) = \max_\pi V_\pi(s), \quad \forall s$$

모든 최적 정책은 동일한 $V^*$를 공유한다. 행동 가치 함수도 마찬가지다.

$$Q^*(s, a) = \max_\pi Q_\pi(s, a), \quad \forall s, a$$

---

### 벨만 최적 방정식 유도

기존 벨만 방정식에 $\pi^*$를 대입하면 아래가 된다.

$$V^*(s) = \sum_a \pi^*(a|s) \sum_{s',r} p(s',r \mid s,a)\left[r + \gamma V^*(s')\right]$$

여기서 핵심 아이디어가 나온다.

최적 결정론적 정책은 **가장 높은 가치를 주는 행동에 확률 1을 부여**한다.

따라서 $\sum_a \pi^*(a|s)$를 $\max_a$로 대체할 수 있다.

**벨만 최적 방정식 ($V^*$)**:

$$\boxed{V^*(s) = \max_a \sum_{s',r} p(s',r \mid s,a)\left[r + \gamma V^*(s')\right]}$$

**벨만 최적 방정식 ($Q^*$)**:

$$\boxed{Q^*(s,a) = \sum_{s',r} p(s',r \mid s,a)\left[r + \gamma \max_{a'} Q^*(s',a')\right]}$$

---

### 기존 벨만 방정식과의 차이

| | 기존 벨만 방정식 | 벨만 최적 방정식 |
| :--- | :--- | :--- |
| 정책 | 특정 $\pi$에 종속 | $\pi^*$ 제거, $\max_a$로 대체 |
| 형태 | 선형 연립방정식 → 풀 수 있음 | **비선형** → 선형대수로 못 품 |
| 목적 | 주어진 정책의 가치 계산 | 최적 가치 직접 계산 |

$\max$ 연산은 선형이 아니기 때문에, 기존 선형대수 방법을 그대로 적용할 수 없다.

---

### 왜 그냥 $\pi^*$를 기존 방정식에 넣으면 안 되나?

$\pi^*$를 모르기 때문이다.

$\pi^*$를 알고 있다면 이미 강화학습의 목표를 달성한 것이다.

→ 대신 벨만 최적 방정식을 기반으로 한 다른 알고리즘 기법을 사용한다.

---

### 핵심 정리

- $V^*$와 $Q^*$는 모든 상태(혹은 상태-행동 쌍)에서 가능한 **최대 가치**이다
- 벨만 최적 방정식은 기존 벨만 방정식에서 $\pi^*$를 $\max_a$로 대체한 것이다
- **비선형**이라 직접 풀기 어렵고, 이후 강의에서 다른 기법으로 해결한다
- $V^*$를 구하면 $\pi^*$를 비교적 쉽게 도출할 수 있다
---
## 3-3 Optimal Value Functions → Optimal Policies
## (최적 가치 함수로 최적 정책 도출하기)

### 핵심 아이디어

$V^*$를 이미 알고 있다면, 최적 정책을 구하는 건 비교적 간단하다.

각 상태에서 **1 스텝 앞을 내다보며** 아래 값을 최대화하는 행동을 고르면 된다.

$$\pi^*(s) = \arg\max_a \sum_{s',r} p(s',r \mid s,a)\left[r + \gamma V^*(s')\right]$$

이건 벨만 최적 방정식에서 $\max$를 $\arg\max$로 바꾼 것과 같다.

---

### 그리드 월드 예시

이전에 배운 그리드 월드에서 $V^*$가 이미 주어진 상황이다.

- 상태 A: 모든 행동 → A'으로 이동, 보상 $+10$
- 상태 B: 모든 행동 → B'으로 이동, 보상 $+5$
- 벽 충돌: 보상 $-1$, $\gamma = 0.9$

특정 상태에서 각 행동의 값을 계산하면:

| 행동 | 보상 + $\gamma V^*(s')$ |
| :--- | :--- |
| 위 | $0 + 0.9 \times 17.5 = 14.0$ |
| 오른쪽 (벽) | $-1 + 0.9 \times 16.0 = 13.4$ |
| 아래 | $0 + 0.9 \times 14.4 = 13.0$ |
| **왼쪽** | $0 + 0.9 \times 17.8 = \mathbf{16.0}$ ✓ |

→ 왼쪽이 최적 행동이다. 동시에 이 값이 $V^*(s) = 16.0$과 일치함을 확인할 수 있다.

**최적 행동이 여러 개인 경우도 있다.**

어떤 상태에서 위와 왼쪽이 둘 다 $17.8
